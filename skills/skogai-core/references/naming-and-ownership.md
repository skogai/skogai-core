<naming_and_ownership>

<overview>
Names should make ownership obvious before the file is opened. A good name states the endpoint's job and keeps competing entrypoints from emerging.
</overview>

<naming_rules>

- Use lowercase hyphenated filenames.
- Use verb-noun names for workflows: `route-information.md`, `write-template.md`.
- Use concept names for references: `routing-framework.md`, `xml-tags.md`.
- Use role names for templates: `routing-file.md`, `workflow-endpoint.md`.
- Avoid vague names like `misc.md`, `notes.md`, `helper.md`, or `guide.md`.

</naming_rules>

<ownership_rules>

- A router owns endpoint selection.
- A workflow owns sequence.
- A reference owns concepts and tradeoffs.
- A template owns repeatable shape.
- A script owns repeatable mechanics.

When two files claim the same ownership, merge, rename, or route one through the other.
</ownership_rules>

</naming_and_ownership>
